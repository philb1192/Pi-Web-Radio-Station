1. Quick Start

Open the web interface in any browser on your local network:

http://<raspberry-pi-ip>:8080

You should see the radio player with your saved stations. Click any station row to start playing, or use the big ▶ Play button.

Tip: Bookmark the page so you can return quickly. On mobile, you can use Add to Home Screen to get an app-like shortcut.

2. Main Controls

▶ Play / ⏸ Pause

The large button below the Now Playing card toggles playback. It is disabled until a station is selected.

Volume Slider

Drag the slider to set the volume from 0 to 100 %. Changes are applied instantly (debounced by 300 ms to avoid flooding the server).

  • In Pi mode — controls the ALSA mixer on the Raspberry Pi.
  • In Browser mode — controls the browser's audio element directly.

🔊 Mute / 🔇 Unmute

Silences audio without stopping the stream. The volume slider position is remembered so unmuting restores the previous level.

Now Playing Card

Shows the current station name. Below it:

  • Stream metadata — artist / track title sent by the station (when available).
  • Reconnect status — appears in amber when the player is trying to recover a dropped stream.

Status Bars

BarMeaning
Green connection barWebSocket is connected to the server.
📱 Bluetooth barA phone is connected via Bluetooth. Shows idle timer if no audio is streaming.
▶ Spotify barSpotify Connect is active. Shows track and artist name.

3. Audio Output Toggle

The 🔊 Pi / 🖥 Browser button in the top-right corner switches where audio plays.

🔊 Pi Mode (default)

Audio plays through the Raspberry Pi's physical speakers or headphone jack via ALSA / mpv. Best for always-on playback — you can close the browser and music keeps playing.

🖥 Browser Mode

Audio streams directly into your browser's audio engine. The Pi's mpv process is stopped. Useful when you want to hear the radio on your laptop or phone.

  • Closing the browser tab stops playback.
  • The volume slider controls the browser's native audio volume.
  • The stream watchdog is disabled in this mode (the Pi cannot monitor browser-side audio).
  • TTS also plays in the browser in this mode.

Tip: Set Default Audio Output in Settings so your preferred mode is restored after a reboot.

4. Stations

Open the 📻 Stations tab to manage your station list.

Health Dots

Each station has a small coloured dot:

  • Green — station is reachable (checked via TCP every 5 minutes).
  • Grey — not yet checked (first check happens 10 s after startup).
  • Red — station is unreachable. Clicking it will skip to the next available station.

Playing a Station

Click anywhere on a station row (except the buttons) to select and play it. The active station is highlighted in purple.

⭐ Default Station

Click the star icon to mark a station as the default. The default station is pre-selected when the page loads. Click the star again to remove the default.

↑ ↓ Reorder

Use the arrow buttons on each row to move a station up or down in the list. Order is persisted to stations.json.

Delete

Removes the station permanently. If it was playing, playback stops.

Adding a Station Manually

Scroll to the Add New Station form at the bottom of the Stations tab. Enter a name and the direct stream URL (must be an HTTP/HTTPS audio stream), then click Add Station.

Note: Common stream URL formats: .mp3, .aac, .ogg, or a plain HTTP endpoint that returns audio data.

5. Online Station Directory

The Search Online Directory panel (inside the Stations tab) connects to the free Radio Browser public API to search ~30 000 internet radio stations.

Searching

Type a station name, genre, or country and press Enter or click Search. Results are sorted by listener votes and show codec/bitrate metadata.

▶ Preview a Station

Click ▶ Preview to audition a station directly in your browser without adding it to your list. The button turns red (■ Stop) while previewing. Clicking Preview on a second result stops the first automatically.

Tip: Preview always plays in the browser regardless of the Pi/Browser output toggle — so you can preview while the Pi continues playing your current station.

Add a Station

Click Add on any result to permanently add it to your station list. The button shows Added and is disabled after clicking.

6. Auto-Reconnect (Stream Watchdog)

The server polls the current stream every 5 seconds. If the stream drops unexpectedly:

  1. It retries the same station up to 3 times. An amber message appears in the Now Playing card.
  2. If all retries fail, it moves to the next station in your list (skipping any already known to be offline based on the health check).
  3. If every station fails, playback stops completely.

Note: The watchdog is inactive in Browser mode — the browser manages its own audio buffering.

7. Text-to-Speech

The 🎙️ TTS tab lets you synthesise speech using the Piper TTS engine installed on the Pi.

Sending a Message

  1. Type your text in the textarea.
  2. Adjust the TTS Volume slider (0–100 %).
  3. Click Speak.

Queue Behaviour

Messages are processed one at a time:

  • Pi mode: The radio stream is paused while speech plays. Once the queue is empty, the stream resumes automatically.
  • Browser mode: Speech audio is synthesised on the Pi and returned as a WAV file played in your browser. Multiple messages queue client-side and play sequentially without interrupting the Pi's audio.

A "N messages queued" counter appears next to the Speak button while the queue is non-empty.

TTS Volume vs. Main Volume

These are independent. TTS volume is set in the TTS tab and in Settings. Main volume controls the radio stream.

Tip: Use the REST API (POST /api/speak) to trigger announcements from Node-RED, Home Assistant, or a cron job — perfect for reminders or door announcements.

8. Bluetooth

CooperStation acts as a Bluetooth A2DP sink — your phone can connect to it like a Bluetooth speaker.

Pairing

  1. On your phone, go to Bluetooth settings and scan for devices.
  2. Select CooperStation.
  3. Accept the pairing on both devices (no PIN required — the agent auto-accepts).

Once paired, the device is automatically trusted so future connections require no confirmation.

Status Bar

When a phone is connected the blue 📱 bar shows the device name and connection state:

  • streaming — audio is actively playing from the phone.
  • idle N min — auto-off in M min — connected but silent; will auto-disconnect after the idle timeout.

Note: Bluetooth audio and radio/Spotify share the same ALSA device. If Spotify or the radio is playing and you connect a phone, audio will mix. Pause the radio first if needed.

9. Spotify Connect

CooperStation appears as a Spotify Connect speaker inside the Spotify app on any device on the same network (or your Spotify account).

Using It

  1. Open Spotify on your phone or computer.
  2. Tap the Connect icon (bottom of player) and select CooperStation.
  3. Press Play — audio comes out of the Pi's speaker.

Status Bar

The green ▶ bar below the connection status shows the current Spotify track and artist. It disappears when Spotify is inactive.

ALSA Conflict Prevention

When you start the radio while Spotify is playing, raspotify is automatically stopped so both don't fight over the sound card. When the radio stops, raspotify is restarted automatically.

Tip: Use Spotify Connect for music you want to control from your phone, and the radio player for background streams you want to run hands-free.

Voice Models

The Voice Models section lets you manage installed models:

  • The Active badge marks the currently selected model.
  • Click Delete to remove a model (you must switch to another model first).
  • Click Download Models ▾ to browse the full Piper voice catalogue (~150 voices).

Downloading a New Voice

  1. Open the Download Models panel.
  2. Filter by language or name using the search box.
  3. Click Download next to any model.
  4. A progress bar appears. The download runs in the background — you can keep using the radio.
  5. Once complete, the model appears in the Active Voice Model dropdown.

Note: Models range from ~30 MB (x_low) to ~200 MB (high). Higher quality requires more CPU on the Pi. The low quality is a good balance.

Saving Settings

Click Save Settings. Changes are written to config.json and applied immediately — no restart required. Default volume also sets the current volume right away.

11. Dark / Light Theme

Click the 🌙 / ☀️ button in the top-right corner to toggle between dark and light themes. Your preference is saved in localStorage and restored on every visit, including preventing a white flash on load.

12. REST API

Every feature is available over HTTP — ideal for Home Assistant, Node-RED, or shell scripts.

Endpoints

MethodPathBody / ParamsDescription
GET/api/status—Full server state as JSON
POST/api/play{"station_id": 1}Play a station by ID
POST/api/pause—Stop playback
POST/api/volume{"volume": 0.75}Set volume (0.0–1.0)
POST/api/speak{"text": "Hello", "volume": 70}Queue a TTS message (Pi output)
POST/api/audio-output{"output": "pi"}Switch audio output (pi or browser)
GET/api/config—Read saved settings
POST/api/configsettings objectSave settings
GET/api/export—Download stations as JSON
POST/api/importstations arrayBulk-import stations
GET/api/directory/search?q=jazz—Search Radio Browser

WebSocket

Connect to ws://<pi-ip>:8080/ws to receive real-time state updates as JSON ({"type":"state_update","state":{…}}) and send actions:

// Play a station
{"action": "play", "station_id": 2}

// Toggle play/pause
{"action": "toggle_play"}

// Set volume
{"action": "set_volume", "volume": 0.6}

// Add a station
{"action": "add_station", "name": "My Station", "url": "http://…"}

Example — curl

# Speak a message
curl -X POST http://192.168.1.x:8080/api/speak \
     -H "Content-Type: application/json" \
     -d '{"text": "Dinner is ready!", "volume": 80}'

# Switch to station 3
curl -X POST http://192.168.1.x:8080/api/play \
     -H "Content-Type: application/json" \
     -d '{"station_id": 3}'

13. Tips & Tricks

Auto-resume after power outage

The server saves playback state to playback_state.json on every play/pause action. If the Pi loses power and reboots, the radio automatically resumes the last-playing station. No action required.

Play a specific station on boot

Star a station (⭐) to make it the default. It will be pre-selected on page load. For true auto-play on boot, make sure the radio was playing when the Pi last shut down — the reboot-resume feature handles the rest.

Use TTS for home announcements

Add this to a cron job or Home Assistant automation:

curl -X POST http://<pi>:8080/api/speak \
     -H "Content-Type: application/json" \
     -d '{"text": "It is now 7 AM.", "volume": 60}'

The radio pauses, speaks, then resumes automatically.

Finding stream URLs

Use the built-in Online Directory search first. If you need a specific station not in the directory:

  • Check the station's website for a "Listen Live" or "Stream" link.
  • Use browser DevTools (Network tab) and look for audio streams while the station's player loads.
  • Sites like radio-browser.info list thousands of verified stream URLs.
Improving audio quality
  • Prefer stations with AAC 128+ kbps or MP3 192+ kbps streams.
  • Download a medium or high quality Piper voice model for better TTS.
  • For Spotify, the config uses 320 kbps — no changes needed.
Accessing from outside your home network

The server runs on port 8080 with no authentication. Only expose it on your local network. For remote access, use a VPN (WireGuard or Tailscale) rather than port-forwarding directly to the internet.

Keyboard shortcuts (browser)

No native shortcuts are built in, but you can use browser extensions or bookmarklets. For example, a bookmarklet to toggle play:

javascript:fetch('/api/play',{method:'POST',headers:{'Content-Type':'application/json'},body:'{}'})
Server log location

When started manually, logs go to stdout. When started via systemd, use:

journalctl -u radio-server -f

Or check /tmp/radio.log if launched with the start script.
